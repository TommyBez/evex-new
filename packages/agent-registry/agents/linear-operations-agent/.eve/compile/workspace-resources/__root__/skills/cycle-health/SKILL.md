# Cycle Health Skill

Use this skill for current-cycle reports or schedule-driven cycle health checks.

Process:

1. Identify configured teams and their current cycle.
2. Read open and recently completed cycle issues.
3. Flag blocked issues, stale P0/P1 work, ownerless work, work added after cycle start, and completed work not closed.
4. Look for owner overload and unresolved dependencies.
5. Keep the Slack report concise and action-oriented.
6. Do not update Linear automatically during scheduled cycle reports.

Output sections:

- Health status
- Key risks
- Issues needing attention
- Scope changes
- Recommended standup topics
