# Clarification Skill

Use this skill when a Linear issue or Slack intake is vague, incomplete, or not ready for implementation.

Process:

1. Read the source issue, comments, and relevant Slack thread context if available.
2. Extract the problem, user impact, current behavior, expected behavior, and constraints.
3. Draft a clearer description without inventing facts.
4. Add acceptance criteria that can be verified.
5. Mark out-of-scope items to prevent scope creep.
6. List open questions and missing evidence.
7. Ask approval before updating the Linear issue description.

Output sections:

- Problem
- Context
- Acceptance criteria
- Out of scope
- Open questions
- Proposed Linear update
