# Severity guide
- SEV1: widespread outage, data corruption, or security event in progress
- SEV2: critical workflow degraded for a meaningful customer segment
- SEV3: isolated or recoverable issue with a workaround

# Upgrade the severity when
- impact is spreading faster than mitigation
- ownership is unclear
- there is any chance of irreversible data loss

# Downgrade only after
- impact has stopped growing
- mitigation is holding
- follow-up owners are assigned
